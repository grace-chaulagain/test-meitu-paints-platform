# Meitu Paints Hostinger Business Deployment

This repository is prepared for Hostinger Business Web Hosting using one managed Node.js web app.

## Chosen Architecture

Use the repository root as the Hostinger app root.

The root app starts `server.js`, which loads the Express backend from `Server/src/server.js`. In production, the backend serves the built Vite frontend from `Frontend/meitupaints/dist` and falls back to `index.html` for non-API routes.

This keeps the app same-origin:

- Website routes: `https://your-domain.com/...`
- API routes: `https://your-domain.com/api/...`
- Refresh cookie: same-origin HttpOnly cookie
- Email links: built from `APP_URL`

## Hostinger hPanel Settings For ZIP Upload

Create the upload package locally:

```bash
npm run package:hostinger
```

Upload:

```text
dist-packages/meitu-hostinger-production.zip
```

In Hostinger hPanel, create a Node.js Web App and choose the ZIP/file upload workflow.

Use these settings:

- App root: ZIP root
- Framework preset: Express.js if available. Use Other only if Hostinger does not auto-detect Express.
- Entry file: `server.js`
- Output directory: leave unset if Hostinger allows it. If hPanel requires a value for Other, use `.`. Do not set this to `Frontend/meitupaints/dist`, because this is not a static-only Vite deployment.
- Build command: `npm ci --omit=dev`
- Start command: `npm start`
- Node.js version: Node 20, 22, or 24

The ZIP already contains `Frontend/meitupaints/dist`, so Hostinger should not run the frontend build for ZIP deployments.

## Required Environment Variables

Set these in Hostinger's Node.js app environment variables:

```bash
NODE_ENV=production
TRUST_PROXY=true
SERVE_CLIENT=true

APP_URL=https://your-domain.com
API_URL=https://your-domain.com
CORS_ORIGIN=https://your-domain.com

MONGO_URI=mongodb+srv://USER:PASSWORD@HOST/DATABASE

JWT_ACCESS_SECRET=replace-with-long-random-secret
JWT_REFRESH_SECRET=replace-with-different-long-random-secret
ACCESS_TOKEN_EXPIRES_IN=10m
REFRESH_TOKEN_EXPIRES_IN=7d
REFRESH_TTL_DAYS=7

COOKIE_SAME_SITE=lax
COOKIE_SECURE=true

SMTP_HOST=smtp.example.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=notifications@example.com
SMTP_PASS=replace-with-smtp-secret
MAIL_FROM="Meitu Paints <notifications@example.com>"
ADMIN_NOTIFICATION_EMAIL=admin@example.com

CLOUD_NAME=cloudinary-cloud-name
API_KEY=cloudinary-api-key
API_SECRET=cloudinary-api-secret
```

Do not set `HOST` on Hostinger. The managed runtime should control host binding.

Do not set `VITE_API_BASE_URL` for this same-origin deployment. The production frontend will call relative `/api/...` routes.

Set `VITE_INQUIRY_ENDPOINT` in the frontend build environment only if the public inquiry form should submit to an external public-safe endpoint such as Google Apps Script. Do not put private API keys or server secrets in any `VITE_*` variable.

## External Services

- MongoDB remains external, recommended through MongoDB Atlas.
- Product/catalog media remains external through Cloudinary.
- SMTP remains external through the configured email provider.
- Factory order PDFs are generated by the backend in memory and attached to email; no persistent local filesystem storage is required.

## Verification After Deploy

Open:

- `https://your-domain.com/`
- `https://your-domain.com/api/health`
- `https://your-domain.com/admin/dashboard`
- `https://your-domain.com/dispatcher/dashboard`
- `https://your-domain.com/dealer/orders`
- a password setup/reset email link generated in production

Deep route refreshes should load the React app instead of returning 404.

## Future Redeploys

For ZIP redeploys, rebuild and upload a fresh archive:

```bash
npm run package:hostinger
```

For GitHub/source deployments instead of ZIP uploads, use the repository root and set the build command to `npm run hostinger:build`.

If environment variables change, redeploy or restart the Node.js app from hPanel.
